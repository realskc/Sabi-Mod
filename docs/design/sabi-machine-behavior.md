# 撒币机行为

相关文件：

- `SabiPawnMachineBlock.java`
- `SabiPawnMachineMenu.java`
- `client/SabiPawnMachineScreen.java`
- `SabiNetwork.java`

## 方块

撒币机的 ID、合成配方、硬度、爆炸抗性和挖掘要求见 `item.md`。

Menu type 是 `sabi:sabi_machine`。

右键撒币机时，服务端打开 `SabiPawnMachineMenu`。

## 交易

典当：

1. 玩家把物品放入典当槽。
2. 服务端检查物品是否允许。
3. 物品进入全局库存。
4. `pawn_price * count` 进入玩家账户。
5. 刷新所有打开的撒币机界面。

赎回：

1. 客户端发送物品和数量。
2. 服务端确认玩家仍靠近撒币机。
3. 服务端检查库存、账户余额和最大堆叠数。
4. 扣账户余额。
5. 从全局库存取出物品，发到玩家背包，背包满则掉落。

购买：

- 只在当前全局库存为 0 时可用。
- 价格为 `pawn_price * 4 * amount`。
- 服务端会重新确认库存仍为 0，防止客户端 UI 过期。
- 购买成功后凭空生成物品。

## 潜影盒快速典当

主列表页快速典当槽里如果是非空潜影盒：

- 点击典当会依次典当盒内可典当物品。
- 盒子本身留在槽内。
- 不可典当物品留在盒内。

主列表页快速典当槽里如果是空潜影盒：

- 点击典当会显示确认页。
- 玩家确认后才典当盒子本身。

设计原因：

- 非空潜影盒通常代表“批量典当里面的东西”。
- 空潜影盒本身也是物品，直接典当容易误操作，所以需要确认。

## 典当槽防丢

典当槽防丢是独立持久化机制，见 `sabi-machine-pending-input.md`。
