# 新增内容清单

本文档面向设计和内容层，列出 Sabi Mod 当前新增的物品、方块和相关规则。更完整的实现细节见 `docs/technical-design.md`。

## 撒币货币

所有实体撒币的最大堆叠数都是 `64`，相邻面额汇率也是 `64`。

| 物品 ID | 中文名 | 英文名 | 账户价值 |
| --- | --- | --- | ---: |
| `sabi:small_sabi` | 小撒币 | Xiao Sabi | 1 |
| `sabi:medium_sabi` | 中撒币 | Zhong Sabi | 64 |
| `sabi:big_sabi` | 大撒币 | Da Sabi | 4096 |
| `sabi:giant_sabi` | 巨撒币 | Ju Sabi | 262144 |

设计要点：

- 实体撒币可通过账户提现获得。
- 实体撒币可在玩家背包或工作台中互相兑换。
- 撒币不可以被撒币机典当。
- 撒币机典当收益直接进入账户；赎回和购买消耗账户余额，不直接消耗实体小撒币。

兑换规则：

- 64 小撒币 -> 1 中撒币
- 1 中撒币 -> 64 小撒币
- 64 中撒币 -> 1 大撒币
- 1 大撒币 -> 64 中撒币
- 64 大撒币 -> 1 巨撒币
- 1 巨撒币 -> 64 大撒币

贴图：

- 小撒币：绿色宝石风格，带黄色标记
- 中撒币：蓝色宝石风格，带黄色标记
- 大撒币：红色宝石风格，带黄色标记
- 巨撒币：紫金宝石风格，带黄色标记

## 个人撒币账户

账户绑定到玩家，不是钱包物品。

入口：

- 在玩家按 E 打开的背包界面中，盔甲/玩家模型区域上方显示账户方框。
- 光标悬停显示完整余额。
- 点击账户方框打开账户界面。

账户界面功能：

- 查看当前余额。
- 输入数量并提现，系统按巨撒币、大撒币、中撒币、小撒币的顺序发放实体货币。
- 一键把背包中的所有实体撒币存入账户。

## 高级红石核心

- 物品 ID：`sabi:advanced_redstone_core`
- 中文名：高级红石核心
- 英文名：Advanced Redstone Core
- 创造模式位置：材料标签
- 用途：撒币机的核心合成原料

合成配方：

```text
R Q R
C E C
R N R
```

材料：

- `R`：红石块
- `Q`：石英
- `C`：红石比较器
- `E`：回响碎片
- `N`：下界合金锭

贴图设计：

- 红石核心外框
- 中央有回响碎片和合金质感核心

## 撒币机

- 方块 ID：`sabi:sabi_machine`
- 物品 ID：`sabi:sabi_machine`
- 中文名：撒币机
- 英文名：Sabi Machine
- 创造模式位置：材料标签
- 硬度：`50.0`
- 爆炸抗性：`1200.0`
- 挖掘要求：钻石镐级别工具

合成配方：

```text
O E O
E C E
O X O
```

材料：

- `O`：黑曜石
- `E`：绿宝石块
- `C`：高级红石核心
- `X`：末影箱

当前功能：

- 右键打开撒币机界面。
- 主界面显示可典当物品网格。
- 玩家可以搜索物品。
- 玩家可以在主界面快速放入任意允许物品并典当。
- 玩家可以点击某个物品进入详情页。
- 详情页可以典当选中物品，也可以输入数量赎回。

交易规则：

- 典当收益直接进入账户，不发放实体小撒币。
- 赎回消耗账户余额。
- 如果某物品当前全局库存为 0，玩家可以花费 `4 * pawn_price` 的账户余额凭空购买该物品。
- 每种物品的典当价优先来自 `base_prices.json` 或 `derived_prices.json`。
- `items.json` 只提供白名单和显示顺序。
- 赎回价不写在配置里，由代码自动计算为 `ceil(pawn_price * 1.2)`。

库存规则：

- 撒币机使用服务器全局库存，任意一台撒币机展示同一批物品。
- 全局库存保存真实 ItemStack。
- 同一种 Item 对应一个栈结构。
- 典当时只尝试和该物品栈顶的 ItemStack 合并。
- 只有同物品且同组件的 ItemStack 才会合并。
- 有耐久、附魔、组件差异的物品不会被错误合并。

贴图设计：

- 黑曜石机器外壳
- 绿宝石正面面板
- 红石指示灯
- 顶部核心槽

## 撒币机物品列表配置

白名单与显示顺序配置文件：

```text
src/main/resources/data/sabi/sabi_machine/items.json
```

当前 `items.json` 是完整白名单，不是自动注册表减黑名单。它也决定 UI 原始顺序，不包含价格字段。

基础价配置文件：

```text
src/main/resources/data/sabi/sabi_machine/base_prices.json
```

这里放无法或不应通过等量代换计算的物品，供人工手动定价。

派生价配置文件：

```text
src/main/resources/data/sabi/sabi_machine/derived_prices.json
```

这里放可计算物品的配方公式。工作台和切石机免费；烧炼按每个输出额外 `1/8` 煤计价；染色类物品使用 `sabi:generic_dye` 通用染料价格；具体染料本身按合成或烧炼配方计价；矿石、蘑菇方块、紫水晶簇等无普通合成公式且本身主要意义是掉落小物的方块可以按时运 III 期望掉落物计价。

生成器还会写入少量手工等价公式，用来表达原版没有固定 recipe 但价格应等量代换的物品。例如水桶/牛奶桶等于桶、鱼桶等于桶加对应鱼、雕刻南瓜等于南瓜、成书等于书与笔、喷溅/滞留药水包含本步酿造材料和烈焰粉摊销、氧化铜制品等于未氧化本体。

`items.json` 配置格式：

```json
{
  "groups": [
    {
      "id": "example_group",
      "items": [
        "minecraft:example_item"
      ]
    }
  ]
}
```

白名单分组规则：

- 同 group 的物品在 UI 中保持相邻，并表达内容层分类；价格来自 `base_prices.json` 或 `derived_prices.json`。
- 只有材质本质相同、形态也相同，只是颜色、纹饰、品种、木种、氧化状态等不同，才应放在同一 group。
- 不同形态不要放同一 group。例如木板、台阶、楼梯应分开。

当前口径：

- 目标是 Java 版生存模式可获得物品。
- 已对照中文 Minecraft Wiki 的不可获得物品清单审计。
- 可疑的沙子和可疑的沙砾在 Java 版口径下保留。
- 撒币货币被明确排除。

当前统计：

- `798` 个 group
- `1363` 个 item
- 价格规则：`183` 个基础价 group，`401` 个基础价 item，`962` 个派生价 item，`1422` 条派生公式项，其中 `21` 条来自时运 III 掉落期望
