# 撒币机典当槽防丢

相关文件：

- `SabiPawnMachinePendingInput.java`
- `SabiPawnMachineMenu.java`

## 问题

撒币机典当槽是菜单内临时容器。

正常按 Esc 或关闭 GUI 时，菜单会走 `removed()`，槽内物品会回到玩家背包。

但玩家退出服务器、连接异常或服务器崩溃时，不能只依赖菜单生命周期，否则典当槽里的物品可能消失。

## 方案

典当槽内容会镜像保存到玩家附件：

```text
pawn_machine_pending_input
```

该附件保存：

- 主界面快速典当槽物品
- 详情页典当槽物品

## 归还规则

- 菜单正常关闭时归还。
- 玩家登出时归还。
- 玩家登录时如果附件里还有挂起物品，自动归还。
- 背包满则掉落。

## 保存策略

典当槽内容变化后，服务端会主动触发一次玩家数据保存。

设计原因：

- 附件跟随玩家数据保存，可以让玩家重进服务器时恢复物品。
- 主动保存可以尽量缩短服务器崩溃时的丢失窗口。

## 维护注意

改 `SabiPawnMachineMenu` 的槽逻辑时，需要确认以下路径仍会同步附件：

- 手动放入物品
- `Shift+Click` 放入物品
- 从典当槽拿回物品
- 典当成功后清空槽
- 非空容器典当后更新容器内内容
